Standard java.io / java.nio compatibility layer.

Java.io support

This package provides:
  • reading data:
    • java.io.InputStream implementation which can be used to read byte [] sequence;
    • java.io.Reader implementation which can be used to read char [] sequence or string sequence;
    • java.io.DataInput implementation which is tweaking the contract to read elementary primitives, byte [] sequence or char [] / string sequence;
  • writing data:
    • java.io.OutputStream implementation which can be used to write byte [] sequence;
    • java.io.Writer implementation which can be used to write char [] sequence or string sequence;
    • java.io.DataOutput implementation which is tweaking the contract to write elementary primitives, byte [] sequence or char [] / string sequence;
  • objects reading and writing:
    • object operation is strictly related to serialization. The abstract format itself does not implement any specific serialization method. However a standard Java serialization may be used to put objects into byte [] sequence.
    • the sztejkat.abstractser library is a recommended method of serializing objects to an abstract format;

Java.nio support

Standard java.nio does not define other "typed" channels than ByteBuffer channels even tough there is a plenty of "typed" buffers (IntBuffer and etc.).

This package does provide "typed" readable and writable channels which can operate on type [] sequences.

Additionally the ByteChannel is supported indirectly via InputStream and can be used to perform a binary mapping of othe types to byte [] sequence. This is however not a recomended method since it makes abstract format to loose it's advantages over a binary format.
