In this chapter I will discuss some rationales behind block operations
  • byte[]:
    • because almost all raw binary data (ie. images) are represented as byte-blocks;
    • any binary data, if necessary can be turned to byte blocks;
    • in XML like format encoding raw binary data as sequence of: 
                               <byte>0x34<byte/>
      is not only a waste of resources but also pointless since human may not benefit from reading raw binary data
  • char[]:
    • because they are used in Strings and Strings may greatly benefit from UTF-8 encoding;
    • because exactly the same primitive operation code may be used to write "signal names" and other data which are internal to format;
  • boolean[] because they can be bit-packed, but bit packing of sequences of bits into sequences of bytes is not trivial when multiple writes are used.
  • other arrays:
    • they can be implemented either over primitive writes or over byte[] at low overhead;
    • they are not used to represent raw data, so in XML like format human may benefit from reading them and encoding them in compact way may be not a best idea;
