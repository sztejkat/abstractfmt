A configuration, patterns and demos for how to construct "contract test suites".

What is "contract test suite"?

This is a set of test cases which are expected to test if implementation of contract X is doing what contract X declares.

In our case it will be always used as a pair of read-write contracts and will use first contract to write data and second to read them back. Thous it may validate if it works correctly.

What is "test case"

A bunch of tests focused on some subject collected in a single test class.

Why not just plain TestXXX class?

  1. because we like to apply tests to different implemenentations of contract;
  2. because tests appears to be huge;
  3. because some contracts may require, just during development, to have fine-tuned subset of tests;

Where is a problem?

Basically in how to force Junit to create specific instance of "device under test" for each tests in many test case classes/
